# nevla model API implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task.
> Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expose nevla's resolved semantic model (symbols, references, call
edges, py boundaries) as a stable public Rust API, `nevla::model::analyze`,
for agent tooling. First consumer: agent-python (`~/projects/agent-python`),
which links nevla as a Cargo path dependency. See agent-python's design doc
(`docs/specs/2026-07-14-agent-python-design.md` there) and nevla ADR 0023.

**Architecture:** A standalone `src/model/` module: structural symbol
extraction from the loaded `Program`, plus a scope-walk resolver with its own
lightweight local tracking. It does not touch typecheck internals and does not
change language semantics, so `language-spec.md` is not affected. All types
serde-serializable so a JSON/gRPC surface falls out later.

## The contract (agreed with agent-python 2026-07-14)

This is the interface agent-python compiles against. Changing it after Phase B
starts requires updating both repos in step.

```rust
// nevla::model
pub struct Pos { pub line: u32, pub col: u32 }        // 1-based, from diag::Span
pub struct SymbolId(pub String);                       // "kind:file:qualified"
pub enum SymbolKind { Module, Function, Struct, Field, Param, Import }
pub struct Symbol {
    pub id: SymbolId,
    pub kind: SymbolKind,
    pub name: String,        // source (unqualified) name, e.g. "Double"
    pub qualified: String,   // loader-qualified, e.g. "util.Double"; root-file symbols are unqualified
    pub file: String,        // as stamped by the loader: filename WITH extension, e.g. "util.nv"
    pub def: Pos,
    pub ty: Option<String>,  // rendered type for functions, None otherwise (skeleton)
    pub is_py: bool,
}
pub enum ReferenceForm { Ident, ModuleMemberCall, StructLiteral }
pub struct Reference {
    pub target: SymbolId,
    pub file: String,
    pub at: Pos,             // the span of the AST node to rewrite (see forms below)
    pub form: ReferenceForm,
}
pub struct CallEdge { pub caller: SymbolId, pub callee: SymbolId, pub at: Pos }
pub struct PyBoundary { pub file: String, pub at: Pos, pub note: String }
pub struct Model {
    pub symbols: Vec<Symbol>,
    pub references: Vec<Reference>,
    pub calls: Vec<CallEdge>,
    pub py_boundaries: Vec<PyBoundary>,
}
pub fn analyze(entry: &Path) -> Result<Model, Vec<Diag>>;  // entry = the .nv entry file
```

All structs/enums derive `Debug, Clone, PartialEq, Serialize, Deserialize`
(`Copy` where trivial, `Eq + Hash` on `SymbolId`, `Default` on `Model`).

**Reference forms and what `at` points to** (verified against `src/parser.rs`
and `src/loader.rs`, 2026-07-14):

- `Ident`: an `ExprKind::Ident` whose string names a module-level fn or
  struct. In the root file these read bare (`compute`); inside an imported
  file the loader has already mangled same-module refs to `stem.name`
  (`util.helper`). `at` is the Ident expr's span. A rename rewrites the
  string's last dotted segment.
- `ModuleMemberCall`: cross-module calls parse as
  `ExprKind::Method { recv: Ident(module), name, .. }`, NOT as a dotted
  Ident. Emitted when `recv` is an Ident matching a file-import stem, is not
  shadowed by a local, and `qualified(module, name)` names a known fn. `at`
  is the Method expr's span. A rename rewrites the Method's `name` field.
- `StructLiteral`: `ExprKind::StructLit { name, .. }` where `name` (bare or
  dotted `mod.Name`) names a known struct. `at` is the expr's span. A rename
  rewrites the name's last dotted segment.

**Known contract limitation (documented, deliberate):** `TypeExpr` carries no
`Span`, so type-annotation positions (`fn f(p Point)`) cannot be emitted as
references. Consumers renaming a struct must rewrite `TypeExpr::Named` by
name-match in affected files; this is sound because nothing shadows a struct
name in type position. `find_callers`/references for structs therefore cover
expression positions only.

**Facts the implementation relies on** (all verified against source):

- `loader::load(entry: &Path)` takes the entry `.nv` file, recurses only into
  `import "x.nv"` paths (the `.nv` suffix is required in source), stamps
  `Decl.file` with filename+extension, and mangles imported modules'
  top-level names and their in-module refs to `stem.name` (scope-aware).
  The root file's names stay bare.
- `diag::Span { line: u32, col: u32 }`, 1-based, no byte offsets.
- AST: `Program { decls }`; `Decl::{Fn(FnDecl), Struct{..}, Import{path, py, span, file}}`;
  `ExprKind::{Ident(String), Call{callee,..}, Method{recv,name,..}, StructLit{name,..}, ...}`;
  locals from `Param`, `StmtKind::Let{names}`, `StmtKind::ForRange{names}`.
- Builtin module calls (`test.eq`, `math.abs`) look like `ModuleMemberCall`
  shape but their stems are not file imports, so target lookup misses them
  naturally. Value-method calls (`xs.append(..)`) have a local recv and are
  skipped by the locals check.
- Export rule (spec 16.3): a capitalized first character exports a name from
  its file module; cross-module access to a lowercase name is a typecheck
  error. Fixture functions called across modules must be capitalized
  (`Double`), and any rename of an exported symbol must keep the
  capitalization (`Twice`, not `twice`).

**Skeleton scope rule (documented limitation):** locals are collected per
function as the union of its params, all `Let` names, and all `ForRange`
names (function-scoped over-approximation, lambdas included). A shadowed name
anywhere in a function suppresses all idents of that name in that function.
This never misclassifies a local as a module symbol; at worst it omits a real
reference, which the consumer's typecheck gate catches. Block-precise scoping
is future work.

fn resolve_finds_cross_file_module_member_call() {
    let prog = load_fixture("model_basic");

    let double = syms.iter().find(|s| s.name == "Double").expect("Double");

- [ ] **Step 2: Verify they fail.**
- [ ] **Step 3: Implement** `src/model/resolve.rs` per the contract's three
  reference forms:
  - Build `targets: HashMap<String, SymbolId>` keyed by `Symbol.qualified`
    for Function and Struct symbols (root-file symbols are bare names).
  - Build `import_stems: HashMap<file, HashSet<stem>>` from
    `Decl::Import { py: false }` (path is already rewritten to a bare stem by
    the loader).
  - Per `Decl::Fn`, collect the function-scoped locals set (params + all
    `Let`/`ForRange` names recursively, including lambda bodies and params),
    then walk the body:
    - `Ident(name)`: skip if in locals; if `targets[name]` hits, emit
      `Reference { form: Ident, at: expr.span }`; if in callee position of a
      `Call`, also emit a `CallEdge`.
    - `Method { recv: Ident(m), name, .. }`: if `m` not in locals and `m` is
      an import stem of this file and `targets[qualified(m, name)]` hits,
      emit `Reference { form: ModuleMemberCall, at: method_expr.span }` and a
      `CallEdge`; do NOT also walk `recv` as an Ident. Always walk args/kwargs.
    - `StructLit { name, .. }`: if `targets[name]` hits, emit
      `Reference { form: StructLiteral, at: expr.span }`; walk field exprs.
    - Recurse through every other `ExprKind`/`StmtKind` variant exhaustively
      (no `_ => {}` on Expr/Stmt containing sub-expressions; let the compiler
      force coverage of Binary/Unary/Index/Slice/List/ListLit/MapLit/Check/
      Conv/Lambda/Field, If/ForRange/ForCond/With/Return/Assign/Let/Expr).
  - py boundaries: `Decl::Import { py: true }` →
    `PyBoundary { note: format!("import py \"{path}\""), .. }`.
  - `caller` for a CallEdge is the enclosing fn's SymbolId
    (`function:{file}:{qualified}`).

pub fn analyze(entry: &Path) -> Result<Model, Vec<Diag>> {
    let prog = crate::loader::load(entry).map_err(|d| vec![d])?;
    let symbols = symbols::extract(&prog);
    let (references, calls, py_boundaries) = resolve::resolve(&prog);
    Ok(Model { symbols, references, calls, py_boundaries })
}
